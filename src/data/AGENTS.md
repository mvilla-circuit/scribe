# AGENTS.md — src/data

The data layer: **React Query** hooks for books, folders, and documents, talking
to Supabase. See the root [`AGENTS.md`](../../AGENTS.md) for repo-wide conventions.

## Area-specific rules

- **Boundaries**: `data` may import from `fonts` and `lib` only. It must not
  reach up into `components`, `editor`, `store`, or `theme`.
- **JSDoc**: exported hooks and types require a description block (no `{type}`
  tags), per the root conventions.
- **Generated types**: query/mutation payloads build on `src/lib/database.types.ts`,
  which is generated by the Supabase CLI — never hand-edit it.

## Testing

- Data-layer tests intercept the **HTTP boundary with MSW** rather than mocking
  the Supabase client. Handlers live in `src/test/msw`; global wiring is in
  `src/test/setup.ts`.
- Tests run against dummy Supabase env values defined in `vite.config.ts`
  (`VITE_SUPABASE_URL=http://supabase.test`, `VITE_SUPABASE_ANON_KEY=test-anon-key`).
  MSW handlers match against that URL, so keep them in sync.
- Use the `render-with-query` helper (`src/test/render-with-query.tsx`) when a
  test needs a React Query provider.
